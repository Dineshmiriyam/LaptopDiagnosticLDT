1. Overview & Architecture

The Laptop Diagnostic Toolkit v6.0 is a USB-portable, single-module PowerShell diagnostic and remediation suite designed for Lenovo ThinkPad fleet management. It runs from any USB drive with no installation required.

Architecture

ComponentFilePurpose
PowerShell ModuleLaptop_Diagnostic_Suite.ps1All 45 functions in one file (~6,000 lines)
BAT LauncherLaptop_Master_Diagnostic.batAdmin elevation, menu UI, function dispatch
ConfigConfig\config.iniAll thresholds, paths, toggles (23 sections)
Module RegistryDiagnosticModules.csv45 rows mapping functions to categories

Key Design Principles

  • Single-module — one PS1 file, no external dependencies
  • USB-portable — all paths use $PSScriptRoot, logs/reports stay on USB
  • ThinkPad-optimized — auto-detects ThinkPad via 3 WMI sources (Model, SystemFamily, ComputerSystemProduct.Version)
  • Non-destructive by default — diagnostic functions are read-only; troubleshooters require Y/N approval per fix
  • Enterprise-ready — Fleet Report exports CSV for aggregation across multiple laptops

2. Quick Start

Step 1: Plug in USB

Insert the USB drive containing the toolkit into the target laptop.

Step 2: Run as Administrator

Right-click Laptop_Master_Diagnostic.bat and select Run as administrator. The BAT file handles UAC elevation automatically if needed.

Step 3: Select an Option

Choose from options 0–48 in the menu. Each option runs a specific diagnostic, troubleshooter, or advanced analysis function.

Step 4: Review Results

HTML reports are saved to Reports\ and logs to Logs\ on the USB drive. Reports open automatically in the default browser.

Tip: For a full system assessment, run options in this order: 1 (Advanced Diagnostic) → 14 (Battery Health) → 17 (Thermal Analysis) → 27 (Security Check). For auto-fix, follow up with options 31–35 (Troubleshooters).
USB Safety: Do not unplug the USB while a function is running. Wait until you return to the main menu. All fixes are applied directly to the laptop — they persist after the USB is removed.

4. Diagnostics (Options 1–5)

All diagnostic functions are read-only — they scan and report without making changes.

Option 1: Advanced Diagnostic

Comprehensive system scan covering disk health (S.M.A.R.T.), memory usage, CPU performance, Windows Update status, event log analysis (errors/warnings from last 7 days), and network configuration. Best for initial triage.

Option 2: BSOD Analysis

Reads C:\Windows\Minidump\*.dmp crash dumps, correlates with reliability records, identifies faulting drivers/modules. Maps known crash-causing drivers (ndis.sys, nvlddmkm.sys, ntfs.sys, etc.) to root causes. Pairs with Option 33 (BSOD Troubleshooter) for auto-fix.

Option 3: Enhanced Hardware Test

Multi-component stress test: CPU load generation (~10 sec), memory allocation/pattern test (100 MB), sequential disk I/O (100 MB), network adapter enumeration, and battery validation. Estimated runtime: 30 minutes.

Option 4: Custom Tests

Select specific test categories to run: storage, memory, display, network, thermal, audio, input devices. Useful for targeted testing when the issue area is already known.

Option 5: Diagnostic Analyzer

Analyzes previously generated log files from the Logs\ folder. Identifies patterns, recurring errors, and trends across multiple diagnostic runs. No admin privileges required.

5. Repair Tools (Options 6–9)

Option 6: Boot Repair

Checks Boot Configuration Data (BCD), runs bootrec/bcdboot, followed by sfc /scannow and DISM /RestoreHealth. Use when a laptop has boot issues or corrupted system files.

Option 7: BIOS Repair

Read-only BIOS version check, age analysis, and update recommendations. Checks against known Lenovo BIOS versions for ThinkPads.

Option 8: Driver Repair

Scans for devices with problem status codes, unsigned drivers, and outdated Lenovo-specific drivers. Can reset individual devices via disable/enable cycle. Pairs with Option 35 (Driver Auto-Update) for automatic updating.

Option 9: Software Cleanup

Scans temp files, browser caches, Windows Update cache, and startup programs. Reports sizes and offers cleanup. Use before Deployment Prep (Option 24).

6. Fleet Management (Options 10–13)

Option 10: Fleet Report

Generates a system inventory report and exports it as CSV for fleet-level aggregation. Captures: serial number, model, OS version, CPU, RAM, disk health, battery status, and more.

Option 11: Verify Scripts

Validates the toolkit itself: checks PS1 module syntax (0 parse errors expected), counts functions (37 expected), verifies BAT launcher dispatch table, and validates config.ini sections.

Option 12: Compatibility Checker

Checks prerequisites: PowerShell version (5.1+ required), .NET Framework, execution policy, admin privileges, CIM/WMI subsystem health.

Option 13: Hardware Inventory

Full CIM hardware enumeration: processor, memory, disk, GPU, network adapters, battery, monitor, USB devices. Exports to CSV.

7. Hardware Tests (Options 14–22)

Option 14: Battery Health

Generates powercfg /batteryreport, analyzes wear percentage (design vs. actual capacity), cycle count, charge rate, and health status. Pairs with Option 28 (Refurb Battery Analysis) for resale scoring.

Option 15: Network Diagnostic

Enumerates all network adapters, checks IP configuration, DNS resolution, latency tests (ping), proxy settings, and VPN adapter status. Pairs with Option 32 (Network Troubleshooter) for auto-fix.

Option 16: Performance Analyzer

CPU/memory/disk benchmarks with bottleneck identification. Reports current resource usage and scores performance relative to expected baselines.

Option 17: Thermal Analysis

30-second temperature monitoring with 5-second sample intervals. Checks fan status where available. Warning at 75°C, critical at 85°C (configurable).

Option 18: Display Calibration

GPU information, current resolution vs. native, monitor details (make/model/size), and dead pixel test guidance.

Option 19: Audio Diagnostic

Enumerates sound devices, checks Windows Audio service status, provides test tone guidance.

Option 20: Keyboard Test

Keyboard device detection, HID driver status check, Lenovo hotkey driver verification. Interactive typing test. Pairs with Option 34 (Input Troubleshooter) for auto-fix.

Option 21: TrackPoint Calibration

TrackPoint and touchpad device detection, pointing device driver check, calibration guidance. ThinkPad-recommended.

Option 22: Power Settings

Lists active power plan, sleep/hibernate/lid settings, and checks for Lenovo Power Manager or Vantage.

8. System Management (Options 23–27)

Option 23: Secure Wipe

Double-confirmation privacy cleanup: removes temp files, browser caches, Windows Update cache, recent file lists. Use before returning a laptop to the pool. Destructive — deleted files cannot be recovered.

Option 24: Deployment Prep

Pre-deployment checklist: disk space, Windows activation status, pending updates, battery health, driver status. Generates a pass/fail checklist for handoff.

Option 25: Update Manager

Checks Windows Update via COM API, detects Lenovo System Update or Vantage presence. Reports pending updates and last update date.

Option 26: BIOS Update

Checks current BIOS version and age. Detects flash utilities in Tools\BIOS\ on the USB. Safety checks: requires AC power, minimum 20% battery.

Option 27: Security Check

Checks: Windows Defender real-time protection, signature age, firewall profiles, BitLocker status, TPM version, Secure Boot, UAC level. Pairs with Option 31 (Security Hardening) for auto-fix.

10. Troubleshooters — Auto-Fix (Options 31–35)

How Troubleshooters Work: Each troubleshooter follows the closed-loop cycle:
Detect Issue → Show Finding → Ask Y/N → Apply Fix → Verify → Log & Report

The technician controls every action via Y/N prompts. No fix is applied without explicit approval.

Option 31: Security Hardening

Pairs with Option 27 (Security Check). Detects and fixes security issues:

Option 34: Input Troubleshooter

Pairs with Options 20 (Keyboard Test) and 21 (TrackPoint Calibration). 8-step check:

v6.0 Improvements (Input Troubleshooter):
  • HID devices with "Unknown" status are now correctly recognized as normal (composite USB devices) and skipped
  • Duplicate HID devices are grouped — one Y/N prompt per device type instead of per instance
  • Sticky Keys detection uses bitmask (bit 2) instead of string comparison, eliminating false positives
  • Error output from device reset operations is cleanly caught and displayed instead of showing raw PowerShell errors
  • Lenovo Hotkey driver fix can now launch Thin Installer directly from USB

Option 35: Driver Auto-Update

Pairs with Option 8 (Driver Repair). Multi-phase driver update:

PhaseAction
1. ScanEnumerate all PnP devices, find: status != OK, driver >12 months old, unsigned
2. CategorizeSafe (auto-update OK), Caution (GPU — needs reboot), Blocked (BIOS/Firmware — never auto)
3. Windows UpdateSearch for driver updates via COM API, install with Y/N per driver
4. Lenovo Thin InstallerIf ThinkPad + Thin Installer on USB → Y/N → silent install

Never Auto-Updated: BIOS and Firmware updates are always blocked from automatic update. The NeverAutoUpdate setting in config.ini controls this.

10b. Advanced Diagnostics (Options 39–48)

These 10 modules provide deep-dive enterprise diagnostics covering POST errors, SMART analysis, TPM health, Windows 11 readiness, and a consolidated enterprise scorecard. All are read-only.

Option 39: POST Error Reader

Scans both System and Application Event Logs for ThinkPad POST/BIOS error codes using a built-in Lenovo error code database (0175–2201). Queries events from providers: Lenovo*, BIOS*, ACPI*, LenovoBiosWmi*. Also checks kernel power events (unexpected shutdowns, BugChecks) and BIOS age. Runs 5 checks total.

Option 40: SMART Disk Analysis

Deep SMART attribute analysis via Get-StorageReliabilityCounter. For each physical disk, checks: read errors, write errors, uncorrectable errors, corrected read errors, start/stop cycle count, temperature, SSD wear level, and power-on hours. Known bad SSD check (UMIS LENSE series on Win11). Also reports free space per logical disk.

Option 41: CMOS Battery Check

Detects CMOS/RTC coin cell battery failure without relying on Win32_Battery (which only reads the main battery). Uses two detection methods:

  • NTP clock drift: Queries w32tm /stripchart to measure time offset from time.windows.com. Warning at >120s, Fail at >3600s.
  • Event Log analysis: Scans Kernel-General (ID 1/12) and W32Time (ID 129/134/142) for time synchronization failures and large time corrections in the last 180 days.
Note: A dead CMOS battery is a $3 hardware fix (CR2032 coin cell replacement) that takes about 5 minutes. This module detects it so the technician knows to physically replace it.

Option 42: Machine Identity Check

Validates machine identity fields critical for fleet management: Serial Number, UUID, Machine Type, and Asset Tag. Detects placeholder values ("To Be Filled", "Default string", all-zeros UUID, all-F's UUID) that indicate failed VPD programming or system board replacement without identity restore.

Option 45: Lenovo Vantage Check

Detects the full Lenovo software stack for enterprise environments (9 checks):

Option 47: Display Pixel Check

Display and GPU health assessment (7 checks):

  • GPU info: Win32_VideoController — driver date, error codes, adapter RAM
  • Display PnP status: Device health via Get-PnpDevice -Class Display
  • GPU TDR events: System log Event IDs 4101/4116
  • Monitor info: WMI monitor parameters (brightness, size, type)
  • Native resolution match: Compares current resolution to VideoModeDescription native resolution — mismatch indicates scaling or driver issue
  • Interactive pixel test: Prompts technician to enter dead pixel count (Read-Host). Evaluates against Lenovo 3-pixel defect threshold (ISO 9241-307 Class II)
  • Backlight bleed test: Prompts technician to rate backlight bleed (None/Minor/Moderate/Severe). Maps to Pass/Info/Warning/Fail. Both prompts gracefully skip in non-interactive mode.

11. Impact Levels & Safety

LevelMeaningOptions
Read-OnlyNo system changes. Safe to run anytime.1–5, 7, 10–22, 24–30, 39–48
CautionMakes system changes with Y/N prompts. Reversible in most cases.6, 8, 9, 31–35
DestructiveDeletes files permanently. Double-confirmation required.23

Safety Features

  • Y/N per fix — troubleshooters ask before every action
  • Invoke-FixAndVerify — every fix is verified after application; result is Pass/Partial/Fail
  • Error suppression — non-terminating PowerShell errors are caught cleanly via $ErrorActionPreference = 'Stop' in the FixAndVerify wrapper, preventing raw red error text from alarming technicians
  • ThinkPad detection — uses 3 WMI sources (Model, SystemFamily, ComputerSystemProduct.Version) for reliable detection
  • Admin elevation — BAT launcher checks for admin and auto-elevates via UAC
  • Logging — every action, approval, and result is logged to Logs\ with timestamps

12. Troubleshooter Workflows

Workflow 1: Laptop Won't Connect to Wi-Fi

  1. Run Option 15 (Network Diagnostic) to identify the issue
  2. Run Option 32 (Network Troubleshooter) → select [1] Wi-Fi Issues
  3. Approve fixes as prompted (enable adapter, flush DNS, reset TCP/IP, etc.)
  4. Review the HTML report for summary

Workflow 2: Laptop Has Frequent BSODs

  1. Run Option 2 (BSOD Analysis) to identify the faulting driver
  2. Run Option 33 (BSOD Troubleshooter) to auto-fix the root cause
  3. If driver update is needed, follow with Option 35 (Driver Auto-Update)
  4. Monitor for 48 hours

Workflow 3: Keyboard Not Working Properly

  1. Run Option 20 (Keyboard Test) to diagnose
  2. Run Option 34 (Input Troubleshooter) to auto-fix
  3. The troubleshooter will check accessibility settings, drivers, and Fn keys
  4. If hardware damage is suspected, the typing test (step 8) will help confirm

Workflow 4: Preparing a Laptop for Resale

  1. Run Option 28 (Refurb Battery Analysis) — check battery score
  2. Run Option 30 (Refurb Thermal Analysis) — check thermals under load
  3. Run Option 29 (Refurb Quality Check) — 10-point QA
  4. Fix any issues: Option 31 (Security) → Option 35 (Drivers)
  5. Final cleanup: Option 23 (Secure Wipe) → Option 24 (Deployment Prep)

Workflow 5: Full System Hardening

  1. Run Option 27 (Security Check) to identify security gaps
  2. Run Option 31 (Security Hardening) to auto-fix all gaps
  3. Run Option 35 (Driver Auto-Update) to patch outdated drivers
  4. Review reports for anything flagged as manual-only

Workflow 6: Enterprise Fleet Assessment

  1. Run Option 48 (Enterprise Readiness Report) for the overall health score
  2. For machines scoring <70, investigate low categories:
  3. — Security issues: Option 43 (TPM) → Option 31 (Security Hardening)
  4. — Storage issues: Option 40 (SMART Disk Analysis)
  5. — Memory issues: Option 46 (Memory Error Log Check)
  6. — Identity issues: Option 42 (Machine Identity Check)
  7. Check Win11 upgrade path: Option 44 (Win11 Readiness Check)
  8. Ensure Lenovo software stack: Option 45 (Lenovo Vantage Check)

Workflow 7: Post-Error Code Investigation

  1. Run Option 39 (POST Error Reader) to identify any logged POST/BIOS errors
  2. If CMOS/RTC errors (0270/0271): run Option 41 (CMOS Battery Check) to confirm
  3. If HDD errors (2100/2110): run Option 40 (SMART Disk Analysis) for detailed disk health
  4. If identity errors (2200/2201): run Option 42 (Machine Identity Check)

  • Tools Directory: Tools\LenovoThinInstaller\ and Tools\VPNInstaller\ for bundled installers
  • [AutoRemediation] Config: New config section with DNS, thresholds, driver update source, blocked categories

v6.0.0 Advanced Diagnostics (10 New Modules)

  • POST Error Reader (39): ThinkPad POST/BIOS error code detection from Event Logs with Lenovo error code database (0175–2201)
  • SMART Disk Analysis (40): Deep SMART attribute analysis: temperature, wear, read errors, uncorrectable sectors, known bad SSD flagging
  • CMOS Battery Check (41): RTC coin cell failure detection via NTP clock drift and Event Log time sync errors (does not use Win32_Battery)
  • Machine Identity Check (42): Serial Number, UUID, Machine Type, Asset Tag integrity validation for fleet management
  • TPM Health Check (43): TPM 2.0 assessment, ROCA vulnerability (CVE-2017-15361) detection, BitLocker integration with Home edition guard
  • Win11 Readiness Check (44): Full Windows 11 hardware compatibility: TPM 2.0, CPU generation (Intel/AMD/Snapdragon), RAM, storage, Secure Boot, UEFI
  • Lenovo Vantage Check (45): Commercial Vantage, Consumer Vantage, System Update, Thin Installer, ImController detection and conflict checking
  • Memory Error Log Check (46): Cross-reference WMD results, WHEA events, BugCheck codes (0x1A/0x4E/0x50/0xD1/0x7A/0xF4) for RAM failure detection
  • Display Pixel Check (47): GPU driver status, TDR events, monitor info, Lenovo 3-pixel defect policy reference
  • Enterprise Readiness Report (48): Master 0–100 weighted health scorecard across 9 categories with lifecycle recommendation and custom HTML scorecard
  • 10 new config.ini sections with configurable thresholds for all new modules
  • Coverage increase: 35 → 45 functions, ~85% of all ThinkPad enterprise issues now covered

v6.0.0 Bug Fixes

  • ThinkPad Detection Fixed: Now checks 3 WMI sources (Model, SystemFamily, ComputerSystemProduct.Version) instead of just Model. Resolves false "Non-ThinkPad" warnings on actual ThinkPads where Model returns machine type code instead of product name.
  • HID Device False Positives Fixed: Input Troubleshooter no longer flags HID devices with "Unknown" status as problems. "Unknown" is normal for composite USB HID endpoints. Only "Error" and "Degraded" statuses trigger the fix prompt.
  • Device Grouping: Duplicate HID devices (e.g., 3x "HID Keyboard Device") are now grouped into a single Y/N prompt per device type, instead of prompting for each instance separately.
  • Sticky Keys Bitmask: Detection changed from string comparison (Flags != "506") to bitmask check (Flags & 4 == 4), which correctly detects only when Sticky Keys is actively ON.
  • Error Suppression: Invoke-FixAndVerify now sets $ErrorActionPreference = 'Stop' so non-terminating CIM/PnP errors are caught cleanly instead of displaying raw red PowerShell error text.
  • Lenovo Hotkey Fix: When the Hotkey driver is missing, the Input Troubleshooter now offers to run Lenovo Thin Installer from USB (if present) instead of just displaying a warning.
